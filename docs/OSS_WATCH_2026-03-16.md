## Update 2026-03-16 (session 2)

### 1. Qwen3 — Modeles disponibles

- **Gamme dense** : Qwen3-0.6B, 1.7B, 4B, **8B**, **14B**, 32B (tous Apache 2.0). Deux modeles MoE : Qwen3-235B-A22B et Qwen3-30B-A3B. Tous disponibles sur Ollama (`qwen3:8b`, `qwen3:14b`, etc.).
- **Performance vs Qwen 2.5** : Qwen3-8B rivalise avec Qwen2.5-14B, Qwen3-14B avec Qwen2.5-32B. Pre-training double (36T tokens vs 18T, 119 langues). Mode thinking/non-thinking integre (raisonnement profond ou dialogue rapide).
- **Vision** : Qwen3-VL existe en 4B et 8B (Instruct + Thinking). Upgrades majeurs en comprehension texte/image, raisonnement spatial, video, et capacites agent. Disponible via `qwen3-vl:8b` sur Ollama.
- **Pour KXKM** : upgrade qwen3:4b vers qwen3:8b = gain significatif pour les personas. Qwen3-VL-8B ouvre la voie a des personas multimodales (analyse image/video).

Sources : [Qwen3 Blog](https://qwenlm.github.io/blog/qwen3/) | [Qwen3-VL GitHub](https://github.com/QwenLM/Qwen3-VL) | [Ollama qwen3-vl](https://ollama.com/library/qwen3-vl)

### 2. MCP (Model Context Protocol) — SDK TypeScript

- **SDK officiel** : `@modelcontextprotocol/sdk` sur npm. Fonctionne sur Node.js, Bun, Deno. Expose server (tools/resources/prompts, Streamable HTTP, stdio) et client (transports, OAuth, helpers).
- **Transport** : le SDK officiel supporte stdio et Streamable HTTP. Pas de WebSocket natif dans le core, mais un SDK tiers (EMQX/MQTT) offre du WebSocket pour le browser. Integration possible en wrappant le transport HTTP dans un relais WebSocket.
- **Integration chat** : le Client MCP expose `listTools`, `callTool`, `listResources`, `readResource`. Un chatbot LLM peut interroger des MCP servers pour enrichir le contexte — exemples fonctionnels dans le repo `examples/`.
- **Pour KXKM** : chaque persona pourrait exposer ses outils via un MCP server. Le WebSocket existant servirait de transport, avec un adapter MCP-over-WS custom. Priorite moyenne — utile quand les personas auront besoin d'outils externes.

Sources : [MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) | [npm @modelcontextprotocol/sdk](https://www.npmjs.com/package/@modelcontextprotocol/sdk) | [MCP Client Guide](https://modelcontextprotocol.io/docs/develop/build-client)

### 3. A2A (Agent-to-Agent Protocol) — Statut

- **Version actuelle** : spec v0.3.0 RC1 (fevrier 2026). Protocole ouvert par Google, 150+ organisations partenaires (Salesforce, SAP, LangChain, etc.). Version production attendue courant 2026.
- **Architecture** : communication HTTPS + JSON-RPC 2.0. Agent Card (JSON) pour discovery des capacites. Objet Task avec lifecycle (creation, progression, completion). Support gRPC ajoute en v0.3.
- **Ecosysteme** : SDK Python disponible, SDK TypeScript/JS en cours. Peu d'implementations production a ce jour — le protocole reste jeune. Google ADK (Agent Development Kit) integre A2A nativement.
- **Pour KXKM** : concept seduisant — chaque persona KXKM pourrait publier un Agent Card et communiquer via A2A. Mais le protocole est encore immature (RC), et le overhead HTTPS/JSON-RPC est lourd pour des personas locales qui partagent deja un bus WebSocket. A surveiller pour interop avec des agents externes, pas pour la communication interne.

Sources : [A2A Specification](https://a2a-protocol.org/latest/specification/) | [Google A2A Blog](https://developers.googleblog.com/en/a2a-a-new-era-of-agent-interoperability/) | [A2A GitHub](https://github.com/a2aproject/A2A)
